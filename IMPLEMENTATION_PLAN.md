## Product summary

An OS-wide utility (macOS + Windows) that lets users select text in any app, press a hotkey, and apply an AI action (Summarize / Edit / Elaborate / Research) to the selection. The result streams into a small floating panel and, on accept, replaces the original selection.

Built with **Tauri v2** (Rust backend, React + TypeScript frontend). Primary LLM provider is **Fireworks.ai**, with **OpenRouter** as automatic fallback on failure.

The defining UX quality is **snappiness**: hotkey to visible panel under 150ms, hotkey to first streamed token under 500ms, never steals focus from the underlying app.

See full design specification in `DESIGN_BRIEF.md` (referenced throughout this plan).

## Naming and conventions

- Rust: standard rustfmt, `tracing` for logs, `thiserror` for error types, `anyhow` only in command layer
- TS: strict mode on, no `any`, ESLint with the Tauri defaults
- Tauri events: lowercase snake_case (`completion_token`, `completion_done`, `completion_error`)
- Tauri commands: lowercase snake_case to match Rust function names

## Phase 4 ŌĆö LLM gateway (Rust)

**Goal:** A clean Rust module that takes (action, text) and streams tokens from Fireworks, falling back to OpenRouter on failure. Cancellable.

**Files:**
- `src-tauri/src/llm/gateway.rs`
- `src-tauri/src/llm/providers.rs`
- `src-tauri/src/llm/prompts.rs`
- `src-tauri/src/llm/sse.rs`
- `src-tauri/src/llm/mod.rs`

**Tasks:**

1. `providers.rs`: define `Provider { Fireworks, OpenRouter }` enum and `ProviderConfig { url, default_model, api_key_storage_key, extra_headers }`. Hardcode the URLs:
   - Fireworks: `https://api.fireworks.ai/inference/v1/chat/completions`
   - OpenRouter: `https://openrouter.ai/api/v1/chat/completions`
2. `prompts.rs`: function `build_messages(action: Action, text: &str) -> Vec<Message>` with the four action prompts from the previous brief (keep them short, lead with "output ONLY")
3. `sse.rs`: parse OpenAI-style SSE chunks. Each chunk is `data: {json}\n\n` with the JSON containing `choices[0].delta.content`. Yield tokens via a stream/channel. Handle the `data: [DONE]` terminator.
4. `gateway.rs`: 
   - `run_completion(action, text, on_token, cancel_token) -> Result<String>`
   - Build messages, build a `reqwest::Client` with HTTP/2 enabled and reasonable timeouts (10s connect, no read timeout because streaming)
   - Try Fireworks first. On any error (network, non-2xx, empty stream), log a warning and fall back to OpenRouter.
   - For each token: append to buffer, call `on_token(token)`, check cancel token
   - On `cancel_token` triggered, return early
5. Reuse a single `reqwest::Client` across calls for connection pooling ŌĆö store it in app state as a `OnceCell`
6. Read API keys from `tauri-plugin-store` (a single JSON file in app data dir). Both keys optional but at least one required.

**Acceptance criteria:**
- Unit test (`cargo test`) with a mocked HTTP server: streaming tokens parsed correctly
- Manual test: call `run_completion` from a small Rust example, see tokens print to stdout
- Force Fireworks to fail (e.g., bad API key) ŌåÆ falls back to OpenRouter automatically with a logged warning
- Cancel token aborts the stream cleanly (no hung connections)

## Phase 5 ŌĆö Wire LLM to the panel (State 2: streaming)

**Goal:** Clicking an action triggers `run_completion`, tokens stream into the panel.

**Files:**
- `src-tauri/src/commands/completion.rs`
- `src-tauri/src/lib.rs`
- `src/components/StreamingView.tsx`
- `src/routes/Panel.tsx`
- `src/lib/tauri.ts`

**Tasks:**

1. `commands/completion.rs`: 
   - `run_completion(action, text)` Tauri command. Spawns a task that calls `gateway::run_completion`, with `on_token` emitting a `completion_token` event with `{ token: String }`. On done, emit `completion_done` with the full text. On error, emit `completion_error`.
   - Store the cancel token in app state so `cancel_completion` can trigger it
   - `cancel_completion()` Tauri command
2. `StreamingView.tsx`:
   - Listens for `completion_token` events, appends to local buffer
   - Renders the streaming text with a blinking caret at the end (CSS animation, simple `::after` content)
   - Status line: "Generating with FireworksŌĆ”" ŌĆö listens for a separate `provider_switched` event from Rust if fallback engages, then updates to "Switched to OpenRouterŌĆ”"
   - Cancel button calls `cancel_completion`
   - On `completion_done`, transition store state to `result`
3. Update Zustand store: action click ŌåÆ set state to `streaming`, call `run_completion`
4. Add the `provider_switched` event in the gateway when fallback happens

**Acceptance criteria:**
- Click any action ŌåÆ tokens stream into the panel in real time
- The first token appears within ~500ms on a decent connection
- Cancel button stops the stream immediately
- If Fireworks fails mid-stream or upfront, status text updates and tokens resume from OpenRouter (the buffer should reset cleanly when falling back from upfront failure; if Fireworks streamed partial tokens before failing, decide policy: easiest is to reset on every provider switch and document that)
- No memory leak on repeated runs (verify with multiple consecutive actions)

## Phase 6 ŌĆö Result view + paste-back (State 3)

**Goal:** After streaming completes, show the result with Accept / Copy / Retry / Back / Dismiss. Accept replaces the original selection in the source app.

**Files:**
- `src-tauri/src/commands/selection.rs` (add `paste_back`)
- `src-tauri/src/input.rs`
- `src/components/ResultView.tsx`

**Tasks:**

1. `input.rs`: `synthesize_paste()` function using `enigo`:
   - Save current clipboard via tauri-plugin-clipboard-manager
   - Write the new result to clipboard
   - Sleep 30ms (let the OS register the clipboard change)
   - Send `Cmd+V` (macOS) or `Ctrl+V` (Windows) via enigo
   - Sleep 150ms (let the paste happen)
   - Restore the original clipboard
2. `paste_back(text)` command: calls `synthesize_paste(text)`. Hides the panel first so focus returns to the source app.
3. `ResultView.tsx`:
   - Renders the final streamed text in a comfortable reading column
   - Collapsible "Show original" section at top, hidden by default
   - Action bar pinned at bottom: primary "Replace" button (or "Copy" if non-editable detected ŌĆö for v1, always offer Replace; if it fails silently, the clipboard still has the result so user can paste manually)
   - Secondary buttons: Copy, Retry (re-runs same action), ŌåÉ Back to actions, Dismiss
   - Keyboard: Enter = Replace, C = Copy, R = Retry, Backspace/ŌåÉ = Back, Esc = Dismiss
4. Panel auto-resizes height based on content up to ~500px max, then internal scroll. Use `set_size` from Tauri or ŌĆö better ŌĆö render in a fixed max-height container with overflow.

**Acceptance criteria:**
- Stream completes ŌåÆ ResultView shows
- Click Replace ŌåÆ panel disappears ŌåÆ selection is replaced in source app with the result
- The user's clipboard before invoking is unchanged after Replace completes (within ~200ms)
- Copy puts result on clipboard and shows brief "Copied" confirmation
- Retry re-runs the same action with the same input
- Back returns to ActionPicker preserving the original selection
- Works in TextEdit, VS Code, browser inputs, Notion, Slack

## Phase 10 ŌĆö Snappiness pass

**Goal:** Hit the target latencies. Hotkey-to-visible < 150ms, hotkey-to-first-token < 500ms.

**Tasks:**

1. **Pre-warm the panel window:** create it hidden at app launch (already done in Phase 1). Verify `show_panel` is a pure `.show().set_focus(false)` call with no rendering or fetch on the critical path.
2. **HTTP keep-alive:** confirm the singleton `reqwest::Client` is configured with `pool_idle_timeout = None` and HTTP/2 prior knowledge for Fireworks (if Fireworks supports it; otherwise let h2 negotiate via ALPN)
3. **Connection pre-warm on app start (optional):** fire a single tiny HEAD/GET to each provider's API root at startup to warm DNS+TLS. Skip if any concerns about cost or telemetry.
4. **Measure:** add a dev-only telemetry overlay (toggleable via `Cmd+Shift+;`) showing recent timings:
   - hotkey_received ŌåÆ panel_visible
   - action_clicked ŌåÆ first_token
   - first_token ŌåÆ completion_done
   Persist last 10 samples in memory for visibility.
5. **Position math:** verify edge-flip logic ŌĆö panel near right edge of screen should flip to appear left of cursor; near bottom should appear above
6. **Cancel speed:** verify Cancel button aborts within one frame (<16ms)

**Acceptance criteria:**
- Measured hotkey-to-visible < 150ms on a typical machine (Apple Silicon or recent Intel)
- Measured first token < 500ms (good network) for a small Llama 8B model
- No visible jank on state transitions
- Pre-warm doesn't visibly delay app startup (< 200ms added to launch)

## Important details and gotchas

These tripped people up while researching this project, calling them out so the agent doesn't have to rediscover:

1. **NSPanel conversion must happen after the window is created but before it's shown.** Order matters ŌĆö do it in the `setup` callback of the Tauri builder after the window exists.

2. **The `get-selected-text` crate uses synthesized Cmd+C as fallback,** which briefly modifies the clipboard. The crate handles save/restore internally but on macOS will play the "alert" sound unless muted. There's a documented workaround in the crate's README ŌĆö apply it.

3. **Tauri events are async and not ordered with command returns.** If `run_completion` emits `completion_token` events from one task while the command returns immediately, ensure the frontend listens before invoking. Standard pattern: register listeners in a `useEffect` before any invoke.

4. **The Fireworks chat completions endpoint requires `stream: true` to actually stream** (obvious but easy to forget). Verify by checking the `Content-Type: text/event-stream` header on the response.

5. **OpenRouter rate limits free models hard.** For the PoC, use a paid model on OpenRouter (Llama 3.1 8B via Together / Fireworks-via-OpenRouter) and assume the user has credits. Document this.

6. **macOS Accessibility permission is per-binary.** During development, Xcode/tauri-dev rebuilds may change the binary identity and re-prompt. Document this for the user; suggest signing the dev build with a stable ad-hoc identity if it becomes annoying.

7. **Don't try to stream tokens via synthesized typing into the source app.** It's slow, looks janky, and breaks autocomplete/spellcheck in the source. Always stream into the floating panel, paste on commit.
